Associative and Commutative

The first idea we want to represent natively in our tool is the associative and commutative properties of basic arithmetic (addition and multiplication). We did this in a way where we represent maths expressions as a stack of blocks, where the blocks on top are treated as the input to the blocks below. To find the value of a block, you would need to resolve the blocks above it before using the value in the current block.

Associative

Associative means that, within the same operation being performed, you can group the inputs in any configuration, as long as the order of the inputs are the same.

(a+b)+c= a+(b+c)

We represented this in our visualisation as the ability to have multiple blocks above addition or multiplication. As well as the actions "Group" and "Ungroup" to add/remove brackets, which are represented as adding/removing layers in the blocks.

Commutative

Commutative means that the operations can operate in a different order.

a+b=b+a

We represented this in our visualisation as the ability to drag elements across the top of commutative blocks.

Combined

The effect of this combined is to be able to do both, but also to be able to group non-adjacent blocks by selecting them (long press), and then applying the group action.

BODMAS/BIDMAS

BODMAS/BIDMAS is all about mathematical grammar. It is a way for us to understand the order that the operations should happen. But there is a subtle difference in which we treat the Brackets and Orders/Indices vs Addition and Subtraction. The difference is due to inverses.

Inverses

Let's first define what Addition and Multiplication can mean in simple terms:
Addition: Let's get n more things
Multiplication: Let's make n copies

Inverses can be roughly thought of as "the same operation, but in the other direction". These would be the definitions in simple terms:
' Subtraction: Let's remove n things
Division: Let's split this into n copies

To represent the "the same operation, but in the other direction", it's useful to simply mark the ones that are inverted. This is where the "unary minus" operation comes in, where we simply add - in front of anything, and we understand that it's to be taken away. This is exactly how we got negative numbers, by using this in front of positive numbers.

To represent the unary inverse, or simply 'marking' which inputs go the other direction, we add a block just below the input.

Multiplication, on the other hand, does not have a "unary division". But to illustrate its similarities to Addition and Subtraction, we have opted to represent the block as a unary inverse. The text view of this will add an additional 1 (the multiplicative identity) in front, so it appears like a binary operation (which it is, but not how we're using it).
